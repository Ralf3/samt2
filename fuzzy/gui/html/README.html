Fuzzy-Interpreter

Method

The program fuzzy was designed as a simple but fast algorithm. It can be used as part of other programs like fuzzy-controller, in simulation systems, etc. Only a few methods are implemented. It can deal with one to three inputs, the membership functions for the inputs can be of type triangular ore trapeze. The output must be from type "singleton", that means crisp value. To use singletons as outputs is not so bad, because it is often sufficient and it is very fast. (you only need to calculate a weighted sum not a integral for the defuzzification) In every rule the inputs are combined by the AND operation (if you need a OR you have to write alternative rules). You can choose the minimum or product method as AND operation. The weight of a rule is calculated by multiplication of the result of the AND operation and a weight factor cf. So you will be able to control the rules in a different way. The aggregation of all outputs of a rule uses the maximum operator. That means that aggregation only occurs ones for each output. The defuzzification process is simple and fast.

Input files and Example

The interpreter will be controlled by a fis file with the following structure:

  • Inputs
  • Outputs
  • Rules

Inputs

input input1 5
member small triangular
-1 10 20
member small_medium triangular
10 20 30
member medium triangular
20 30 40
member medium_hot triangular
30 40 50
member hot triangular
40 50 65

The inputs must begin with the keyword "input", the name of the input and the number of membership functions. There are up to three inputs possible. The membership function can be of type trapeze or triangular. For the trapeze type there are 4 numbers (the lower left point, the upper left point, the upper right point and the lower right point), for the triangular we only need 3 numbers.

Outputs

outputs temperature 5
output cold 10
output cold_medium 25
output medium 35
output medium_hot 45
output hot 60

In that example are 5 outputs for temperature from cold to hot.

Rules

The rules combine the inputs with the outputs by numbering. In our example we have only one input. That means that the rule consists only of three columns: input, output, cf
rules 5
0 4 1.0
1 3 1.0
2 2 1.0
3 1 1.0
4 0 1.0

This rule base is very simple. You can change the system by putting rules in the rule base with different cf-factors. An other way to change the system is to change the shape of the membership function. Please check it.

Install

The system consists of following files:

  • input.h input.cpp
  • output.h output.cpp
  • member.h member.cpp
  • rule.h rule.cpp
  • fuzzy.h fuzzy.cpp
  • main.cpp
  • Makefile
  • example files: 3.fis 2.fis wanne.fis wanne1.fis, wanne.txt as inputs
  • README.html
  • fuzzy.gif

Get the program: Fuzzy_Interpreter

Please unpack the fuzzy.tgz with:

tar -zxvf fuzzy.tgz

and type make

The main is only one example for the fuzzy. You can put in our system and use it under terms of the GPL. It works fine under Linux but I think it should be used under any kind of Unix.

Future

Maybe that the program is useful for you, so I plan to develop a graphical user interface to build the rule base. Please let me know it. I use that program because it is simple and fast, so I'm able to do many thousand fuzzy inferences per second. The program is easy to extend for different membership function like bell shape functions, but this would be lead to a slower program. If you need more then three inputs you should split your problem in parts and combine it by a new fuzzy system.
